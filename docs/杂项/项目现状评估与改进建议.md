## 3. 目前达到的阶段（重点结论）

当前项目已从“想法验证”进入“可运行工程原型”阶段，具体表现：

1. 主流程全链路跑通：读取报告 -> 抽取 -> 规则质控 -> LLM 复核 -> 分级 -> 文书生成 -> 输出结果。
2. 核心降级策略存在：在模型不可用时不会全链路中断。
3. 规则与生成已打通：存在可执行的“先质控、后生成”门控逻辑。
4. API 具备流式步骤输出：可用于前端演示和过程观测。

但尚未进入“可量化验收阶段”：

1. 缺实验体系，无法证明“有质控比无质控更好”。
2. 缺真实标注数据，无法证明模型与规则在真实中文业务上的效果。
3. 缺人工反馈闭环，无法形成持续改进的数据飞轮。

## 4. 关键问题清单（按优先级）

## 4.1 P0（高风险，优先处理）

### P0-1 中文影像抽取能力不足（你指出的问题成立）

现象：

1. RadGraph 对中文报告经常返回空实体，代码里已有该异常路径判断。
2. 触发后转入 fallback 解析。

影响：

1. 影像类抽取质量不稳定，关系信息可信度不足。
2. 下游影像质控与分级会受误差放大影响。

证据：

1. modules/ner_re/imaging_ner_re.py（_extract_with_radgraph 中对 empty entities 的处理）
2. modules/ner_re/imaging_ner_re.py（extract_imaging_ner_re 的 fallback 逻辑）

### P0-2 影像 fallback 规则过于简化

现象：

1. 词表规模小。
2. 病变与解剖关系采用“病变节点 x 解剖节点”笛卡尔连接，缺上下文定位。

影响：

1. 关系抽取可能产生大量伪关系。
2. 影像质控误报/漏报并存。

证据：

1. modules/ner_re/imaging_ner_re.py（_fallback_imaging_parse）

### P0-3 缺少效果评测与对比实验框架

现象：

1. 当前测试主要是单元/流程功能测试。
2. 没有标准化离线评测脚本和实验配置矩阵。

影响：

1. 无法回答立项核心问题：质控是否真实提升质量。
2. 无法比较不同基座模型和是否人工介入的收益。

证据：

1. tests/test_main_pipeline.py
2. tests/test_imaging_ner_re.py
3. 仓库中缺少 modules/evaluation 或 experiments 目录

## 4.2 P1（重要缺口，需在一个月内补齐）

### P1-1 规则库覆盖不足

现象：

1. 指标范围目前仅少量关键指标。
2. 影像 lesion_anatomy_map 规模较小。

影响：

1. 规则质控覆盖面不足，行业可用性受限。

证据：

1. config/qc_rules.json

### P1-2 人工复核与反馈闭环缺失

现象：

1. API 主要提供流程运行，未提供人工裁决回写。
2. 生成阶段能返回 pending_issues，但没有结构化修复建议生成器。

影响：

1. 无法把“专家纠错”沉淀为可学习数据。

证据：

1. api.py
2. modules/case_generation/medical_record.py

### P1-3 可观测性不足

现象：

1. 运行摘要仅统计总量与 degraded_count。
2. 缺 source 分布、失败原因分布、步骤耗时等关键指标。

影响：

1. 难以定位性能瓶颈和质量波动原因。

证据：

1. main.py（run_summary 输出结构）

## 6. 改进路线图（可执行）

## 6.1 两周内（止血与可测）

目标：先让系统“可观测、可对比、中文影像不失控”。

动作：

1. 为影像抽取新增“中文优先路径”：LLM 结构化抽取（病变、部位、关系）优先，RadGraph 作为英文备选。
2. 升级 fallback：扩大词表 + 增加最邻近窗口匹配，避免全连接关系。
3. 建立基础评测脚本：固定测试集，输出抽取覆盖率、降级率、问题检出率。
4. 扩展 run_summary：增加 source_distribution、failure_reasons、step_latency。

验收标准：

1. 中文影像样本 degraded 率下降或在 degraded 下关系质量可控。
2. 每次运行自动生成可比较的质量摘要。

## 6.2 一个月内（能力补齐）

目标：补齐立项中的“评测对比 + 人工参与 + 数据闭环”。

动作：

1. 新建实验配置：有无质控、有无人工、基座模型 A/B。
2. 增加人工反馈接口：问题确认、修正内容、置信度回写。
3. 新增修复建议模块：针对缺失/矛盾给结构化纠正建议。
4. 扩充规则库：指标范围与影像映射逐步扩容，并配套回归测试。

验收标准：

1. 形成至少一轮完整对比实验报告。
2. 能看到人工参与带来的净收益（例如误报下降）。

## 6.3 长期（体系化升级）

目标：形成稳定生产能力。

动作：

1. 建立中文医学标注数据资产（NER/RE/QC）。
2. 引入专门中文医学 IE 模型并与规则协同。
3. 建立持续评估看板（质量、时延、成本、失败路径）。

验收标准：

1. 关键指标可长期追踪并支持发布决策。
2. 模型或规则更新有量化收益证明。

---

## 7. 最小可执行任务包（建议立刻开工）

1. 新建中文影像抽取器（LLM 结构化方案）并接入影像主路由。
2. 重构 fallback 关系生成逻辑（去笛卡尔全连接）。
3. 新建 evaluation 脚本与实验配置文件（最少 4 组实验）。
4. 扩展 run_summary 输出字段并加错误原因分类。
5. 新增修复建议生成模块并挂到病历输出。
6. 新增人工反馈 API（问题确认与修正回写）。
7. 扩充 qc_rules 基线词典并增加回归测试。

并行建议：

1. 抽取增强（任务 1、2）可并行于评测建设（任务 3、4）。
2. 人工闭环（任务 5、6）可并行于规则扩容（任务 7）。

---

## 8. 结论

你当前项目已经完成了“先质控后生成”的工程主干，方向是对的。

当前最大短板不是流程是否存在，而是“中文影像抽取质量 + 评测闭环缺失”。其中你提到的 RadGraph 中文不适配，是现阶段影像质量瓶颈的核心成因。

建议优先从“中文影像抽取替代路径 + 可量化评测体系”两条线同时推进，先把质量与证据建立起来，再做规则扩容与模型迭代。
